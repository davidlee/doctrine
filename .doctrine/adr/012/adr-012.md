# ADR-012: Dispatch integration topology: isolated coordination worktree, class-routed projection, preserved code branches

> Status: **accepted.** Split out of the SL-064 design under RV-023 F-5 (the
> topology is framework-significant — ADR-011 precedent — too large to bolt onto
> ADR-006 as a "D8 amendment"). This ADR owns the *topology + projection +
> routing*; the pure-placement amendments stay on ADR-006 (see §Boundary).
> **Acceptance gates SL-064's plan** (cf. SL-056 G1 / ADR-008). OQ-A/B/C are
> resolved and normative here; OQ-D is **reclassified to an SL-064 plan-gate**
> (not an ADR-acceptance blocker) with the residual risk consciously accepted
> (User ruled (a)). Adversarial trial: RV-025 (+ RV-023 on the spine).

## Context

`/dispatch` today runs the orchestrator's coordination branch *on the shared
session working tree* (`main`): the funnel commits land there directly; "conclude"
only flips status + GCs debris. ADR-006 D8 pins the coordination *branch* but is
silent on its *working tree*, and "globals land on main by convention, not
mechanism." That breaks under (a) foreign writers on `main`, (b) concurrent
dispatches contending with each other, (c) any team posture where unreviewed code
must not touch trunk — the SL-060 retrospective
([[mem.system.dispatch.orchestrator-on-shared-main-contention-cost]]) showed the
whole contention cost landing orchestrator-side.

The fix is not just *placement* (an isolated coordination worktree — that part is
an ADR-006 D8 refinement). It is a new **integration topology**: how dispatch
output materialises, persists, and *escapes into the repo* for review. That is
branch-flow doctrine, and ADR-006 D1 holds doctrine to *detection, not
prescription*. Making the framework opinionated about integration flow is a
conscious tightening that deserves its own decision — this ADR.

## Decision

**We will give dispatch an opinionated, configurable integration topology:**

1. **Isolated coordination worktree per run.** The orchestrator's coordination
   branch `dispatch/<slice>` runs in its own dedicated worktree (always-on), not
   the session tree. It is the funnel's sole write target and the run's durable
   SSoT + crash-recovery root. Slice-scoped and stable across handover-resume;
   concurrent same-slice dispatch is refused at creation.

2. **Delta routing on a temporal + dependency boundary** (OQ-B). The clean
   code-vs-intent split (worker source-only deltas vs orchestrator-written
   `.doctrine/` knowledge, separated by R-5) is the *directional default*; the
   load-bearing seam underneath it is **temporal/dependency**, in four buckets:

   | Bucket | Examples | Projection |
   |---|---|---|
   | **Prior governance** | accepted ADR / design / plan | reviewed + landed *before* impl; not projected by dispatch |
   | **Impl bundle** | code + AC evidence + phase notes + design amendments + status caused by impl | ships **together** as one review unit |
   | **Slice-orthogonal knowledge** | durable memory/pattern *not* dependent on the unmerged code | may project ahead independently |
   | **Runtime coordination** | phase sheets, transient dispatch state | stays on `dispatch/<slice>`; never trunk material |

   **Runtime classifier (the decision rule, not just policy prose):** an
   impl-time write **holds with the impl bundle by default**; it projects ahead as
   slice-orthogonal **only** when explicitly marked orthogonal by a separate,
   deliberate act. Ambiguous-by-default ⇒ holds. This forecloses both bad
   outcomes: "code + a wall of markdown = a poor PR" *and* trunk `.doctrine` claims
   describing code not yet accepted.

3. **Branch/target roles** (framework names roles; project binds refs —
   opinionated defaults, overridable):
   - `dispatch/<slice>` — coordination SSoT.
   - `phase/<slice>-NN` — preserved per-worker **code deliverables**, integrable
     individually.
   - `review/<slice>` — the **named home for the impl-bundle review unit** (the
     "leave-for-review" target; *not* trunk).
   - `candidate/<slice>/<label>` — ordinary mutable **interaction branches** for
     post-execution audit review, repair, close-target admission, and experiments.
     Candidates are produced from evidence refs, not instead of them.
   - `edge` — optional standing local aggregate.

   ("feature" is reserved for the PRD Requirements-aggregation sense — never a
   branch name.) **Harness synthesis rule:** on arms that do **not** return a
   per-worker fork branch — the Claude `Agent` arm (ADR-011), where the worker
   delta lands directly onto `dispatch/<slice>` — the per-phase deliverable branch
   is **cut from `dispatch/<slice>` at sync time**, so `phase/<slice>-NN` is a
   universal deliverable regardless of arm.

4. **Integration-sync = a two-stage downstream projection, via a verb, never
   auto-landing** (OQ-A). A `doctrine`-internal sync verb (in-process git,
   golden-pinnable, not skill-prose) reads the completed `dispatch/<slice>` and
   projects outward **at conclude** (the controlled sync point — never per-batch,
   which would re-create the contention this topology removes).
   - **Stage 1 — prepare-review** (`dispatch sync --prepare-review`, name TBD at
     plan): materialise the reviewable refs — `review/<slice>` for the impl bundle,
     `phase/<slice>-NN` for code — and the journal. **No trunk write.**
   - **Stage 2 — integrate** (only after audit passes, §Decision 5): optional
     projection to trunk / `edge`, using an admitted candidate OID when the
     candidate workflow is active.
   - **Routing default:** intent projects to **`review/<slice>` by default, never
     trunk-by-default.** Trunk projection is **opt-in, fast-forward-only,
     expected-tip-guarded**; never force-push, never auto non-ff;
     conflict/moved-target ⇒ **report, never auto-resolve** (ADR-006). Timing =
     conclude. IMP-043 re-anchor lives here. Default config shape (project binds):
     `intent_target = "review"` (alt `"trunk_ff_only"`), `intent_timing =
     "conclude"`, `code_target = "phase_branches"`, `aggregate_target = "edge"`
     (optional).
   - **Recovery contract — normative (the F-1 penance):** a **journal committed to
     `dispatch/<slice>` before any external ref mutation**, recording the source
     OID and, per step: `target_ref`, `expected_old_oid`, `planned_new_oid`,
     `applied_new_oid`, `status`. **Every ref update is a compare-and-swap on
     `expected_old_oid`.** Replay recomputes intended outputs from the source OID;
     if `target == planned_new_oid` ⇒ mark verified (no-op); if `target` differs
     from **both** `expected_old_oid` and `planned_new_oid` ⇒ **refuse + report a
     moved target**; never force-push, never auto-resolve. This restores a
     D7-analog crash-recovery property *across* projection: after a crash the
     coordination branch can explain projection state and replay is safe. Contract
     is normative here; its implementation + proof land in SL-064.

   - **Candidate interaction/admission (SL-068 amendment):** candidate
     materialisation is the only place this topology permits a 3-way
     auto-resolution step. Candidate creation is explicit, creates a normal branch
     from a chosen base with zero-OID CAS, no-ff merges the chosen evidence source,
     and records immutable `source_oid`, `base_oid`, and `merge_oid` on
     `dispatch/<slice>`. It never mutates trunk, `edge`, `review/*`, or `phase/*`.
     Admission records an immutable `admitted_oid` after validating provenance:
     the Doctrine-created `merge_oid` has the recorded base/source parents, and
     `merge_oid` is an ancestor of the admitted tip. Close/integrate targets the
     admitted OID, never the current value of a mutable candidate ref. If the close
     target moved so the admitted OID no longer fast-forwards it, close refuses
     with a moved-target reason and requires a superseding candidate on the new
     base; it never creates, updates, rebases, merges, or repairs a candidate at
     close time.

5. **Audit runs between the two stages, against the reviewable materialisation**
   (OQ-C). The RV review verbs refuse on a worktree fork
   ([[mem.pattern.dispatch.rv-verbs-refuse-on-worktree-fork]]) and the
   coordination worktree is removed at conclude. Ordering: **conclude → stage-1
   prepare-review → audit from the parent/root context against the prepared
   `review/<slice>` + `phase/<slice>-NN` refs, or the candidate refs derived from
   them → reconcile → stage-2 integration.** Audit therefore gates the **actual
   review units before they integrate**, not an upstream source after publication.
   Audit failure **blocks** trunk/`edge` integration but **preserves**
   `dispatch/<slice>` + `phase/*` + `review/*` + `candidate/*` for remediation.
   The sync verb does not run audit; audit is never driven from inside the
   coordination worktree.

6. **D1 is consciously tightened, not "preserved."** The framework now opines on
   integration topology. Configurable refs preserve repo-policy flexibility, but
   doctrine now owns an opinionated integration topology. Legitimacy rests on that
   configurability + the opt-out being absent-by-design (the in-place path is
   retired, not kept as a hazardous parallel funnel).

7. **V1 close payload remains code by default (SL-068 amendment).** Review-surface
   candidates may be sourced from `review/<slice>` and include the impl bundle so
   humans and external reviewers have a safe surface. Close-target candidates use
   the phase-chain code tip by default, preserving the pre-SL-068 trunk payload
   semantics. An impl-bundle close-target is an explicit/later mode, not the
   default v1 trunk path.

## Boundary — what stays on ADR-006

These are *placement/identity* refinements to the existing worktree posture, not
topology, and remain ADR-006 amendments:
- **D8** — coordination-branch working-tree placement (dedicated worktree).
- **D2a** — orchestrator may run in a *linked* worktree; write-permission rests on
  marker-absence (positive signal), not `!is_linked_worktree`; `env
  DOCTRINE_WORKER` must not leak.
- **D9 addendum** — markerless coordination-tree creation variant.

The **positive coordination-tree marker** (the strictly-safer identity redress) is
*not* an ADR-006 amendment here — it touches the owner-locked D2a positive-signal
model and is deferred as OQ-D / **IMP-065** (see §Decisions formerly open).

## Consequences

### Positive
- Contention surfaces (SL-060 #1 dirty foreign index, #2 inline `.doctrine`
  collisions) unreachable by construction.
- Team feature-branch/PR posture is first-class: unreviewed code never touches
  trunk; intent is visible on `review/<slice>` (slice-orthogonal knowledge may
  project ahead).
- Concurrent dispatches mutually isolated.
- Projection is crash-recoverable from `dispatch/<slice>` (the normative CAS
  journal), closing the F-1 gap the spine had to disclaim.
- Audit gates the exact refs and admitted candidate OIDs that integrate, before
  they integrate.
- Candidate refs provide a safe, self-describing interaction surface for review
  repairs and experiments without rewriting evidence refs.

### Negative
- One cold provision build per run (per-worktree target), amortised; stings a
  quiet-solo-small-slice.
- New framework surface (a two-stage sync verb, normative projection journal/CAS
  replay, branch lifecycle) — the cost of owning the altitude honestly.
- **Review surfaces multiply.** A slice may leave `dispatch/<slice>`,
  `phase/<slice>-NN`, `review/<slice>`, `candidate/<slice>/<label>`, and optional
  `edge` refs. Lifecycle + cleanup UX must make each branch's **purpose and
  integration state obvious** — an owned cost of the topology, designed-for in
  SL-064 and extended in SL-068.
- **Residual identity gap (OQ-D / F-2), consciously accepted.** v1 rests
  coordination-tree write-permission on **marker-absence**, indistinguishable from
  an *unstamped* worker, with the ADR-006 D2b residual fence (R-5 import belt +
  IMP-052 post-spawn check + env-worker-on-main catch + repo-local bwrap-no-push)
  as **defence-in-depth, not a proof** — it does *not* prove coverage of the full
  Orchestrator verb class (`gc`/`sync`). Bounded by the OQ-D plan-gate below; the
  real close is the positive marker (IMP-065).

### Neutral
- Repo-local concerns (rtk, cargo, `CARGO_TARGET_DIR`, bwrap) remain ADR-008
  instances, layer-walled out
  ([[mem.pattern.design.product-not-compromised-by-project-local-ops]]).

## Decisions formerly open

- **OQ-A — RESOLVED, normative** (Decision 4). Two-stage projection; intent →
  `review/<slice>` by default, trunk opt-in ff-only + expected-tip CAS; journal
  committed to `dispatch/<slice>` before any ref mutation; CAS replay
  (no-op-if-planned / refuse-if-diverged). Closes RV-023 F-1.
- **OQ-B — RESOLVED, normative** (Decision 2). Temporal/dependency boundary, four
  buckets, default-hold-with-bundle classifier; orthogonal projects ahead only by
  a deliberate mark.
- **OQ-C — RESOLVED** (Decision 5). Audit between stages, from parent/root,
  against the prepared review refs; failure blocks integration, preserves branches.
- **OQ-D — RECLASSIFIED + DEFERRED** (User ruled (a)). **Not an ADR-acceptance
  blocker.** Markerless coordination creation ships as a **transitional
  assumption** with the D2b fence as defence-in-depth. It becomes an **SL-064
  plan-gate**: the plan MUST (i) **restrict Orchestrator-verb invocation
  (`fork`/`import`/`gc`/sync) to the trusted orchestrator path** until a positive
  marker lands, and (ii) include **impersonation tests**. The positive
  coordination marker (a D2a amendment) is **IMP-065**. Residual risk consciously
  accepted.

## Verification

- Black-box golden for the two-stage sync verb.
- Orchestrator never writes the session `main` tree mid-run (contention #1/#2
  unreachable).
- **Projection recovery (OQ-A), crash matrix:** (a) crash after journal-planned,
  before ref update; (b) crash after ref update, before journal-verified; (c)
  target moved between plan and apply; (d) replay is a no-op when target already
  equals `planned_new_oid`; (e) replay refuses + reports when target diverged from
  both. Every ref update asserted CAS-guarded.
- **Default routing:** intent lands on `review/<slice>`, not trunk, with default
  config; trunk projection only under explicit ff-only policy and refuses a
  non-ff/moved trunk.
- **OQ-B classifier:** an unmarked impl-time write holds with the impl bundle;
  only an explicitly-marked orthogonal write projects ahead; runtime coordination
  never leaves `dispatch/<slice>`.
- **Harness synthesis (M3):** on the Claude arm (no fork branch), `phase/<slice>-NN`
  is cut from `dispatch/<slice>` at sync and is a valid deliverable.
- **Audit fixture (OQ-C):** RV verbs refuse inside the coordination worktree;
  succeed from parent/root against the prepared `review/<slice>`; a failed audit
  blocks trunk/`edge` sync and preserves all deliverable branches.
- **Candidate provenance/admission (SL-068):** candidate create records
  `source_oid`, `base_oid`, and `merge_oid`; candidate admit rejects tips that do
  not descend from the recorded `merge_oid`; close uses `admitted_oid`, not a
  mutable candidate ref.
- **No close-time merge (SL-068):** close appends/replays a normal CAS journal
  row with `planned_new_oid = admitted_oid`; if the admitted OID does not
  fast-forward the target, close refuses and requires a superseding candidate.
- **Candidate/ref guards (SL-068):** candidate creation uses zero-OID CAS for new
  refs; raw evidence refs are unchanged after candidate create/admit; write-ish
  commands refuse from worker-mode and raw evidence worktrees.
- Concurrent same-slice dispatch refused at creation.
- **OQ-D plan-gate (not a fence proof):** SL-064's plan restricts Orchestrator-verb
  invocation to the trusted orchestrator path and carries impersonation tests; the
  D2b fence is asserted only as defence-in-depth, *not* as proof it catches `gc`/
  `sync` impersonation (that close is IMP-065).
