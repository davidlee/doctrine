# ADR-012: Dispatch integration topology: isolated coordination worktree, class-routed projection, preserved code branches

> Status: **proposed → accepting.** Split out of the SL-064 design under RV-023
> F-5 (the topology is framework-significant — ADR-011 precedent — too large to
> bolt onto ADR-006 as a "D8 amendment"). This ADR owns the *topology +
> projection + routing*; the pure-placement amendments stay on ADR-006 (see
> §Boundary). **Acceptance gates SL-064's plan** (cf. SL-056 G1 / ADR-008). The
> four open questions are now closed (OQ-A/B/C resolved; OQ-D consciously deferred
> with fence — §Decisions formerly open).

## Context

`/dispatch` today runs the orchestrator's coordination branch *on the shared
session working tree* (`main`): the funnel commits land there directly; "conclude"
only flips status + GCs debris. ADR-006 D8 pins the coordination *branch* but is
silent on its *working tree*, and "globals land on main by convention, not
mechanism." That breaks under (a) foreign writers on `main`, (b) concurrent
dispatches contending with each other, (c) any team posture where unreviewed code
must not touch trunk — the SL-060 retrospective
([[mem.system.dispatch.orchestrator-on-shared-main-contention-cost]]) showed the
whole contention cost landing orchestrator-side.

The fix is not just *placement* (an isolated coordination worktree — that part is
an ADR-006 D8 refinement). It is a new **integration topology**: how dispatch
output materialises, persists, and *escapes into the repo* for review. That is
branch-flow doctrine, and ADR-006 D1 holds doctrine to *detection, not
prescription*. Making the framework opinionated about integration flow is a
conscious tightening that deserves its own decision — this ADR.

## Decision

**We will give dispatch an opinionated, configurable integration topology:**

1. **Isolated coordination worktree per run.** The orchestrator's coordination
   branch `dispatch/<slice>` runs in its own dedicated worktree (always-on), not
   the session tree. It is the funnel's sole write target and the run's durable
   SSoT + crash-recovery root. Slice-scoped and stable across handover-resume;
   concurrent same-slice dispatch is refused at creation.

2. **Two delta classes, routed on a *temporal* boundary** (OQ-B). The clean
   code-vs-intent split (worker source-only deltas vs orchestrator-written
   `.doctrine/` knowledge, separated by R-5) is the *directional default*; the
   load-bearing seam underneath it is **temporal**, in three buckets:
   - **Prior gate** — scope + design: reviewed and landed on trunk *before*
     implementation. Not part of projection at all (code + a wall of markdown is
     a poor PR; the design review is its own earlier gate).
   - **Impl-time bundle** — code **plus** the intent-drift that arose *during*
     implementation (notes, slice-coupled memory, plan adjustments, design
     amendments, AC evidence). Ships **together** as one reviewable unit at the
     implementation review gate; slice-coupled intent does **not** project ahead
     of its code.
   - **Slice-orthogonal knowledge** — durable cross-cutting memory not coupled to
     this slice's review. The *only* intent that may project to trunk
     contemporaneously.

3. **Three branch roles** (framework names roles; project binds refs —
   opinionated defaults, overridable): `dispatch/<slice>` (coordination SSoT),
   `phase/<slice>-NN` (preserved per-worker code deliverables, integrable
   individually), `edge` (optional standing local aggregate). ("feature" is
   reserved for the PRD Requirements-aggregation sense — never a branch name.)

4. **Integration-sync = downstream projection, via a verb, never auto-landing**
   (OQ-A). A `doctrine`-internal sync verb (in-process git, golden-pinnable, not
   skill-prose) reads the completed `dispatch/<slice>` and projects outward **at
   conclude** (the controlled sync point — never per-batch, which would re-create
   the contention this topology removes).
   - **Routing/timing:** projection pushes **review-pending refs** (the kept
     `phase/<slice>-NN` and the slice-coupled impl-time bundle) toward the
     configured trunk-tracking target *for review*. It **never force-pushes and
     never auto non-ff merges to trunk**; the final trunk landing is a
     human/external gate. Conflict or moved-trunk ⇒ report, never auto-resolve
     (ADR-006). IMP-043 re-anchor lives here, relocated from the per-batch funnel.
   - **Recovery contract (the F-1 penance):** projection carries a **journal
     committed to `dispatch/<slice>`** (the durable SSoT). Each step records the
     target ref, the **observed precondition tip**, the planned op, and a
     completion marker — **intent written before the op, completion after**.
     Replay is **idempotent**: every step is "ensure target ref contains commit
     set X", verified by **patch-id presence** (`git cherry`,
     [[mem.pattern.dispatch.landed-oracle-needs-import-receipt]]), so re-running a
     completed step is a no-op. On resume: read journal → observe live target
     tips → execute each incomplete step whose precondition still holds; if a
     target advanced past its recorded precondition (a concurrent mover) ⇒
     **report, never auto-resolve.** This restores a D7-analog crash-recovery
     property *across* projection — to the extent the journal+replay is built and
     tested in SL-064. This ADR fixes the **contract**; it does not claim a proof.

5. **Audit runs after projection, from the parent tree** (OQ-C). The RV review
   verbs refuse on a worktree fork
   ([[mem.pattern.dispatch.rv-verbs-refuse-on-worktree-fork]]) and the
   coordination worktree is removed at conclude. So the ordering is **conclude →
   projection (review-pending refs) → audit from the parent tree against the kept
   `dispatch/<slice>` → reconcile → final trunk landing gated by audit + human.**
   The sync verb does not run audit; audit is never driven from inside the
   coordination worktree.

6. **D1 is consciously tightened, not "preserved."** The framework now opines on
   integration topology. Legitimacy rests on configurability (every target a ref)
   + the opt-out being absent-by-design (the in-place path is retired, not kept as
   a hazardous parallel funnel).

## Boundary — what stays on ADR-006

These are *placement/identity* refinements to the existing worktree posture, not
topology, and remain ADR-006 amendments:
- **D8** — coordination-branch working-tree placement (dedicated worktree).
- **D2a** — orchestrator may run in a *linked* worktree; write-permission rests on
  marker-absence (positive signal), not `!is_linked_worktree`; `env
  DOCTRINE_WORKER` must not leak.
- **D9 addendum** — markerless coordination-tree creation variant.

The **positive coordination-tree marker** (the strictly-safer identity redress)
is *not* an ADR-006 amendment here — it touches the owner-locked D2a
positive-signal model and is deferred as OQ-D / **IMP-065** (see below).

## Consequences

### Positive
- Contention surfaces (SL-060 #1 dirty foreign index, #2 inline `.doctrine`
  collisions) unreachable by construction.
- Team feature-branch/PR posture is first-class: unreviewed code never touches
  trunk; intent stays visible (slice-orthogonal knowledge projects immediately).
- Concurrent dispatches mutually isolated.
- Projection is crash-recoverable from `dispatch/<slice>` (the journal contract),
  closing the F-1 gap the spine had to disclaim.

### Negative
- One cold provision build per run (per-worktree target), amortised; stings a
  quiet-solo-small-slice.
- New framework surface (a sync verb, projection journal/replay, branch
  lifecycle) — the cost of owning the altitude honestly.
- **Residual identity gap inherited honestly (OQ-D / F-2):** v1 rests
  coordination-tree write-permission on **marker-absence**, indistinguishable from
  an *unstamped* worker — and SL-064 widened that blast radius to the Orchestrator
  verb class. Held by the **ADR-006 D2b residual fence** (R-5 import belt + IMP-052
  post-spawn check + env-worker-on-main catch + bwrap-no-push), pinned by a §6
  verify-item. The positive-marker redress is split to **IMP-065**.

### Neutral
- Repo-local concerns (rtk, cargo, `CARGO_TARGET_DIR`) remain ADR-008 instances,
  layer-walled out ([[mem.pattern.design.product-not-compromised-by-project-local-ops]]).

## Decisions formerly open (now closed)

- **OQ-A — RESOLVED** (Decision 4). Routing: never-auto-trunk; review-pending
  refs projected at conclude. Recovery: projection journal on `dispatch/<slice>`
  + idempotent patch-id replay; report-not-resolve on a moved target.
- **OQ-B — RESOLVED** (Decision 2). Temporal three-bucket boundary; "intent →
  trunk contemporaneously" holds only for slice-orthogonal knowledge.
- **OQ-C — RESOLVED** (Decision 5). Audit after projection, from the parent tree.
- **OQ-D — DEFERRED with fence** (ruled (a)). v1 ships the marker-absence model +
  the ADR-006 D2b residual fence; the positive coordination-tree marker (a D2a
  amendment) is split to **IMP-065**. Acceptance does not force a D2a redesign.

## Verification

- Black-box golden for the sync verb.
- Orchestrator never writes the session `main` tree mid-run (contention #1/#2
  unreachable).
- Projection is idempotent / crash-recoverable from `dispatch/<slice>`: replaying
  a partially-applied projection is a no-op on completed steps and reports on a
  moved target (OQ-A journal contract).
- Slice-coupled intent ships *with* its code (one review unit); only
  slice-orthogonal knowledge projects ahead (OQ-B).
- Audit runs from the parent tree against `dispatch/<slice>` after projection,
  never from inside the coordination worktree (OQ-C).
- Concurrent same-slice dispatch refused at creation.
- **D2b fence covers Orchestrator-verb impersonation (F-2):** an unstamped worker
  in a coordination-shaped tree invoking `fork`/`import`/`gc`/sync is caught by
  the R-5 belt / IMP-052 post-spawn check / jail — assert it (until OQ-D / IMP-065
  replaces absence with a positive marker).
