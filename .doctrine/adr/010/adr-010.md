# ADR-010: Relation modelling: unify the pure-ref class, keep payload edges typed

## Context

Doctrine's entities reference each other, but the *storage* of those relations is
bespoke per kind, surveyed during SL-046 design and code-verified for IMP-034:

- **slice** (`src/slice.rs`) — `Relationships{specs, requirements, supersedes}`, all
  `Vec<String>`, parsed-inert.
- **governance** (`src/governance.rs`) — `Relationships{supersedes, superseded_by,
  related, tags}`, `Vec<String>`; stores the reciprocal `superseded_by` (an ADR-004
  violation, separately tracked as IMP-032), parsed-inert.
- **backlog** (`src/backlog.rs`) — `Relationships{slices, specs, drift, needs}` plus
  `after: Vec<AfterEdge{to, rank}>` and `triggers: Vec<Trigger{globs, note}>`; live.
- **spec** (`src/spec.rs`) — lineage `descends_from`/`parent` as `Option<String>` on
  the `Spec` struct; `members{requirement, label, order}` in a separate
  `members.toml`; `interactions{target, kind, notes}` in `interactions.toml`.

There is **zero** shared relation type; cardinality is mixed (`Vec` vs `Option`),
structure is mixed (plain ref vs ranked vs labelled vs glob-payload), aliveness is
mixed. IMP-034 asks whether **all** of this should refactor to one uniform generic
schema (`[[relation]] kind="…" target="…"` + optional attributes).

Two adjacent decisions constrain the answer:

- **ADR-004** — relations are stored outbound-only; reciprocity is derived, never
  authored on the target. Any uniform schema must compose with this.
- **SL-046** (the relation *reader*, design locked) already unifies the *read* of
  the **reference/lineage subset** behind a per-kind `relation_edges` accessor
  emitting a uniform `RelationEdge{label, target}`. It is read-only and does not
  cover `needs`/`after`/`triggers`/`tags`/`interactions`.

The forcing function is **SL-048**, which is blocked on a project-global
relation-governance decision and wants a cross-kind `link`/`relate` *write* verb.
SL-046's read seam cannot supply a shared writer; only a shared storage shape can.

An interrogation (IMP-034) with an external adversarial pass (codex GPT-5.5)
established that relations partition into two classes with opposite answers: pure
reference edges, where a uniform block is a net win and unlocks the shared writer,
and payload-bearing edges, where genericising the typed structure is a net loss.

## Decision

We will **unify the pure-reference relation class to a single authored block and
keep payload-bearing relations as bespoke typed structures.** Specifically:

1. **Partition relations into two classes.**
   - *Pure-ref* (no edge-local payload): slice `{specs, requirements, supersedes}`;
     backlog `{slices, specs, drift}`; governance `{supersedes, related}`; spec
     `{descends_from, parent}`; spec `interactions` (already a generic
     `{target, kind, notes}` row).
   - *Payload* (typed edge-local data): backlog `after{to, rank}`,
     `triggers{globs, note}`; spec `members{requirement, label, order}`.

2. **Unify the pure-ref class** onto one generic `[[relation]] kind="…" target="…"`
   authored block, shared across every kind that carries such edges. This yields one
   parser, one validator, one renderer, one extraction seam, **and** a single
   cross-kind `link`/`relate` writer — the deliverable SL-048 needs.

3. **Keep the payload class typed.** `after`, `triggers`, and `members` carry
   edge-local data consumed as typed values (ordering rank, trigger globs, mobile
   membership `label`/`order`). A generic attribute bag would push required-field
   guarantees out of the parser into ad-hoc validators and read worse — a net loss.

4. **The relation-label vocabulary is authoritative in code, not in this prose.** A
   single source of truth (a `RelationLabel` enum plus a per-source-kind legal-set
   table declaring which labels are legal and which target-kinds they admit) drives
   parser, validator, writer, *and* the SL-046 reader in lockstep. ADR-004 is
   reaffirmed: the vocabulary admits **outbound** labels only; no inverse label
   (`superseded_by`) is authorable, so the generic block cannot regress ADR-004's
   outbound-only invariant. This code-level constraint is what keeps a generic array
   from being *weaker* than the typed fields it replaces.

5. **Migration is bounded to the pure-ref blocks.** No payload relation is touched.
   The governance `superseded_by` removal is owed regardless of this ADR
   (IMP-032 / SL-048) and is not new cost attributable here.

6. **State the contract in principle; do not bind it to unshipped API names.**
   SL-046 is `design`, not shipped. This ADR fixes the partition, the
   code-authoritative vocabulary, and outbound-only — not the concrete
   `relation_edges`/`RelationEdge` identifiers, which lock only once that seam
   survives contact with code.

Rejected alternatives: **(A) full uniform storage** — genericises the payload class,
a confirmed net loss. **(B) no storage change, unify only the read contract** —
leaves four bespoke write paths and cannot give SL-048 its shared `link` writer.

## Consequences

### Positive

- One shared cross-kind relation *writer* (`link`/`relate`) becomes possible — the
  SL-048 prerequisite that the read-only SL-046 seam could not satisfy.
- Uniform parse / validate / render / extraction for the pure-ref class, with the
  legal-edge vocabulary enforced in code rather than convention.
- ADR-004 is strengthened, not weakened: the outbound-only invariant is enforced by
  the vocabulary at the type level instead of relying on per-kind field naming.
- Bounded, well-scoped migration; the typed payload cases keep their guarantees.

### Negative

- A corpus-wide migration of the pure-ref blocks (slice, governance, backlog pure-ref
  subset, spec lineage) — every affected authored TOML and its renderer changes.
- A second relation modelling *idiom* persists (generic block + typed payload tables);
  the system is uniform within each class, not globally.
- The `RelationLabel` vocabulary is new surface that must stay in lockstep with the
  reader's overlay labels and the validator — a maintenance coupling to hold.

### Neutral

- Aliveness (inert vs live) is unchanged by this ADR — it is a reader/verb property
  settled by SL-046/SL-048, orthogonal to storage shape.
- Spec `interactions` moves from a bespoke file into the pure-ref block; its
  target-kind validation is unchanged in substance.

## Verification

- The shared `[[relation]]` parser/validator rejects any label outside the
  per-source-kind legal set, and rejects every inverse/derived label (no
  `superseded_by` authorable) — unit-tested at the vocabulary boundary.
- The SL-046 reader's overlay labels and this vocabulary derive from the one code
  source; a test asserts they cannot diverge.
- Behaviour-preservation gate: `backlog order` stays byte-identical (payload
  relations `after`/`needs` untouched); existing per-kind suites stay green.
- The migration of each pure-ref block is verified by round-trip read of every
  affected entity via `doctrine <kind> show`.
