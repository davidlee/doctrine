# ADR-016: Relation intent as a closed role dimension

## Context

RFC-003 triaged a holistic relation-model review (CHR-024) into four axes. Axis B ‚Äî
"overloaded edge intent" ‚Äî is the core: the `specs` / `requirements` / `related` labels
conflate distinct *intents*. A live census found ~25 of ~113 entity‚Üíentity reference
edges asserting an intent their label gets wrong (a risk cannot *implement* a spec; an
improvement only *bears on* one; `related` is a degenerate catch-all whose peer reading
is the minority). The defect is a noun verbing: `specs` names the target *kind*, never
the verb ‚Äî and the missing verb *is* the role.

RFC-003 asserts no canon; adoption requires a ratifying ADR (RFC ¬ß Outcome). Per the
routing decision for SL-149, that ADR is folded into the slice's `/design` and authored
here. It composes with **ADR-010** (unify the relation contract + write seam, storage
bespoke) and **ADR-004** (relations stored outbound-only, reciprocity derived); it does
**not** supersede either ‚Äî ADR-010's storage/seam decisions remain true, this adds an
intent dimension on top.

## Decision

**1. Structure/intent split.** A relation's durable *structural shape* ‚Äî the **label**
(sources, target-class, tier, graph-effect, inbound semantics) ‚Äî is separated from its
*contextual intent* ‚Äî a closed **role**. The work‚Üícanon family collapses into one
`references` label refined by `{implements, scoped_from, concerns}`. Target validation
re-keys from `(source, label)` to `(source, label, role)`: type safety is preserved, the
target gate simply relocates from label to role. `governed_by`, `related`, `part_of`,
`supersedes`, `exclusive_with` stay distinct labels.

> **Deciding principle (the ruling).** Use a *separate LABEL* when the distinction
> changes **structure, graph-effect, or inbound semantics**; use a *ROLE facet* when it
> only refines the **intent** of an otherwise structurally-identical edge.

- Corollary ‚Äî `related` **stays a label**: it is symmetric + neutral, its reciprocal is
  itself, where every `references` role has a distinct directional inbound verb. Symmetry
  is an inbound-semantic difference ‚Üí structural ‚Üí label, not role.
- Corollary ‚Äî `reviews` is **not** a role (see ¬ßruling below).

**2. Derivable-not-relational law.** Do not encode in the relation what is derivable from
entity state or facets. **Coverage** (‚Üí `validate` / `/close`), **temporal
planned-vs-done** (‚Üí target lifecycle status), and **altitude** (‚Üí target facet) are
*projections*, not labels. `slices` stays one edge; planned/done reads off status.

**3. Graph-effect in the consumer.** Whether an edge gates, evicts, or scores is consumer
policy (priority overlay / `/close` / `validate`), not a property of the relation
(consistent with IMP-047). Corollary: cordage overlay allocation stays **label-keyed**,
not `(label, role)` ‚Äî `references` is one overlay-backed label regardless of role;
cordage stays vocabulary-unaware.

### Vocabulary rulings (RFC-003 divergence ‚Äî recorded explicitly)

- **`bears_on` ‚Üí renamed `concerns`.** The RFC's `bears_on` term is jargony and weak;
  the design dialogue settled on `concerns`. The rename is deliberate; recorded here so
  the RFC‚ÜîADR vocabulary divergence is explicit, not silent.
- **`reviews` folds into `concerns` (dropped as a role).** `reviews` has no structural
  distinction from `concerns` ‚Äî work ‚Üí any target, evaluative/relevance intent, no gate.
  The heavyweight, dispositioned review is already the first-class **RV `reviews`
  label**, which is untouched. The review-in-prose cases SL-145 deferred are *aboutness*
  ("this work concerns that artefact") and are captured by `references(concerns)`; B does
  **not** launder a distinct evaluative edge into `concerns`. A dedicated non-RV
  evaluative role is YAGNI (zero live instances); if a genuine evaluative-vs-relevance
  distinction ever earns its keep it is added then. The "loss" is a distinction that was
  never instantiated, not a downgrade of existing data.

### Scope boundaries (named, deferred ‚Äî not decided here)

- Whether work can `implements` an **ADR** ‚Äî `governed_by` stays the ADR relation for
  now; the likelier resolution is that ADRs impose REQs and the REQ is the implemented
  thing.
- **Non-entity-target edges** (memory / file / glob / vec) ‚Äî IMP-012, IDE-015.
- **Decomposition `part_of` + altitude facets** ‚Äî Axis D sibling spec. The
  `scoped_from`-vs-`part_of` boundary is held by definition + the target-kind gate
  (backlog-only targets), not a structural invariant; cleanup deferred to Axis D.
- **`exclusive_with`** and the **`influences` / relation-planes** question
  (directionality √ó valence) ‚Äî modelled in RFC-003, demanded by no current edge.

## Consequences

### Positive

- Each reference edge carries a real, type-checked intent; the ~25 mis-asserting edges
  become expressible correctly. The target gate is strengthened (re-keyed on role), not
  weakened.
- One `references` label replaces the work‚Üícanon family; new intents are a closed-enum
  code change (deliberate cost ‚Äî keeps the vocabulary auditable).

### Negative

- Adding any future role is a code change (closed enum, by design). Migration of the live
  corpus is a one-shot out-of-band hard cut (no shipped `migrate` verb), with hand-triage
  of ambiguous rows.

### Neutral

- Inbound rendering gains a role dimension; cordage overlay keying is unchanged
  (label-keyed). The ADR-004 `superseded_by` reverse carve-out is untouched.

## Verification

Enforced by SL-149's phase plan: the two-level grammar lives in `RELATION_RULES` with
table invariants (one rule per `(source, label, role)`; each `(source, label)` wholly
roleful or wholly roleless); lockstep tests audit source-sets and exact `(label, role)`
coverage; the migration carries a role-assignment oracle (asserts assignment, not merely
edge survival) plus a reviewed disposition artifact for every ambiguous row.
