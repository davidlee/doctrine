# ADR-008: Project-local jail build isolation and worker confinement for parallel dispatch

## Context

This ADR is **project-local to doctrine-the-repo** — how doctrine dogfoods the
parallel worktree dispatch defined by ADR-006 inside its own bubblewrap jail. It
is downstream of ADR-006 and prescribes nothing for consuming projects.

It also **discharges ADR-006 D2b** (raw-tree confinement), which D2b explicitly
deferred to "sandbox/harness work (ADR-008)". A generic harness cannot confine a
worker to its worktree (observed: Claude Code creates the worktree but lets the
agent write to main). Doctrine's jail *is* a sandbox, so doctrine can solve here
what the framework layer cannot.

Project-local facts that force the decisions:

- `CARGO_TARGET_DIR=/home/david/.cargo/doctrine-target-jail` is a **single shared
  absolute path** (flake.nix). Two worktrees on different branches sharing it
  cargo-lock-serialize *and* thrash — each branch's source invalidates the
  other's fingerprints.
- `~/.cargo/bin/doctrine` is **read-only** in the jail, so `cargo install` is
  impossible in-jail regardless.
- `CARGO_BIN_EXE` is baked at compile time; e2e tests spawn
  `$CARGO_TARGET_DIR/debug/doctrine` (a shared target gives a stale/foreign
  binary — the known stale-`CARGO_BIN_EXE` spawn-fail).
- justfile recipes are env-driven (`cargo build/test/install`), no hardcoded
  paths.

## Decision

- **D-B1 — Per-worktree `CARGO_TARGET_DIR`.** Each worktree gets its own target
  dir, nested under the jail-redirect root and keyed by worktree/branch
  (`…/doctrine-target-jail/wt/<branch>`), **set at worker spawn** rather than
  baked in the flake. This restores real build parallelism, eliminates
  cross-branch fingerprint thrash, and gives each worktree a correct
  `CARGO_BIN_EXE`. Cost: one full target dir per concurrent worktree (disk) plus
  cold builds.

- **D-B2 — No `cargo install` mid-dispatch.** Installation is impossible in-jail
  (ro `~/.cargo/bin`) and is not attempted. Workers and their tests run the
  per-worktree **build-target** binary; the ro `~/.cargo/bin/doctrine` remains the
  stable entrypoint. There is no install race because there is no install.

- **D-B3 — Per-worker bwrap confinement (spike-first).** The intended discharge of
  ADR-006 D2b: run each worker in a nested bwrap that **rw-mounts only its
  worktree and its target dir, ro everything else (including main)**, so a worker
  *physically cannot* write to main — OS-enforced, not prompt-enforced. We will
  **spike this first (timeboxed)**; nested bwrap may be easier than feared. If the
  spike shows it is too costly, **back out to D-B1 + the ADR-006 D2a CLI guard**
  and accept the residual raw-edit risk for doctrine's own (largely solo)
  dogfooding, leaving D2b deferred. D-B3 depends on D-B1 (the confined worker
  needs its target dir rw).

- **D-B4 — `sccache` deferred (noted).** A shared compilation cache would offset
  D-B1's per-worktree cold-build disk/time cost. Deferred until cold builds
  actually cause pain; recorded here so the option is not rediscovered.

- **D-B5 — Keep the flake minimal.** Per-worktree env is set at worker spawn, not
  baked; the justfile is unchanged (already env-driven). The only added surface is
  a `dispatch-worker` bwrap profile (D-B3).

## Consequences

### Positive

- doctrine-the-repo becomes a **reference implementation** of safe parallel
  worktree dispatch, and (if D-B3 lands) of *true* worker confinement — proving
  ADR-006 D2b is solvable wherever a sandbox exists.
- Per-worktree targets (D-B1) make parallel build/test genuinely concurrent
  instead of cargo-lock-serialized, with correct e2e binaries.

### Negative

- Disk grows with concurrent worktrees (N target dirs); a worktree cap or D-B4
  may be needed under disk pressure.
- D-B3's nested bwrap adds plumbing; the spike may fail and force the weaker
  fallback (D-B1 + D2a), leaving D2b deferred.

### Neutral

- Nothing here is exercised until `/dispatch` is implemented and run on doctrine
  itself; this ADR is governance-ahead-of-need so the eventual implementation
  slice inherits the decisions.
- Host-vs-jail mount-path separation is preserved: per-worktree dirs nest under
  the existing jail-redirect root.

## Verification

- D-B1: two parallel worktree builds run without cargo-lock contention, each
  producing a binary its own e2e tests spawn successfully (no stale-`CARGO_BIN_EXE`).
- D-B3 (if landed): a worker process, given a write to a path outside its
  worktree (e.g. main's `.doctrine`), is denied by bwrap — asserted by an
  attempted out-of-tree write failing.
- D-B2: dispatch performs no `cargo install`; the build-target binary is the one
  under test.
