# ADR-011: Harness-agnostic orchestrator spawn interface and per-harness capability altitude

## Context

**Mechanism in prose is the design smell** (SL-056 thesis, surfaced by SL-055's
holistic skills review). The worktree/dispatch CLI today owns three small read
verbs (`provision`, `check-allowlist`, `branch-point-check`); everything else —
the fork-creation ladder, the import funnel, worker-mode enforcement, build
isolation, GC — lives as **prose asking an LLM to replay git command sequences.**
That prose is token-heavy (the two fattest skills), **untestable** (no golden pins
a prose ritual), **fail-open** (a skipped step refuses nothing), and
**harness-coupled by accident** (written against Claude's `Agent` affordances). A
CLI verb is identical under claude/codex/pi by construction — each mechanism moved
into the binary makes skills shorter **and** more agnostic in the same move.

The trust-bearing core is **who controls the worker's worktree and identity**. The
keystone is **the orchestrator owning fork-or-mark + provision + per-worktree env
emission, and stamping a disk marker** into the worker's worktree before the worker
runs: **identity rides disk — the one medium every harness has — not a process env
seam, which not every harness has** ([[mem.pattern.dispatch.spawn-backend-harness-agnostic-no-free-env-seam]]).
claude's `Agent` tool has no env channel, and `claude -p` is Anthropic-API-billed
(not subscription) and harness-specific — so neither an env seam nor a subprocess
spawn can be a *required* element of a harness-agnostic framework (SL-056 Charge
XIII). For claude the only viable backend is the in-session `Agent` tool, which
exposes no env seam and no exec wrapper.

This ADR fixes the **contract** every harness honours and the **altitude** each can
reach. It **references ADR-006 and does not amend it** — the ADR-006 D2a/D5/D9
amendments are a separate, spike-gated step (SL-056 G2/PHASE-03), authored only
after the O3 spike (PHASE-02) validates the claude path.

## Decision

The mechanism/concession line falls between **what the binary does** (create-or-mark
+ provision + per-worktree env *contract* emission — harness-identical) and **how
the worker is spawned** (harness-shaped → prose, selected by the `/dispatch-*`
router). The binary side is the contract; the spawn side is a per-harness concession.

### D1 — Worker identity is an orchestrator-stamped disk marker (agnostic floor)

Worker-mode is a property of the **worker**, signalled by a **disk marker the
trusted orchestrator stamps before the worker runs** — presence-only, no contents,
at the withheld-tier path `<root>/.doctrine/state/dispatch/worker`. The guard is:

```
worker_mode(root) := (is_linked_worktree(root) && marker_present(root))  // PRIMARY, agnostic
                     OR env DOCTRINE_WORKER set                          // codex/pi worker-on-main catch
guard (in run(), before a write-classed OR Orchestrator/Hook-mint Command):
    if worker_mode(root): refuse(verb)   // names the verb
```

The marker is **primary and harness-agnostic**; `DOCTRINE_WORKER` is a **codex/pi
optimisation, not the identity** — its one job is to catch the **worker-on-main**
hazard (a harness that drops the worker on the coordination root, where no marker
exists). Solo `/execute` sets neither signal → writes freely: **mode, not location,
decides** (ADR-006 D6a).

### D2 — The orchestrator owns the trusted act; the worker writes source only

The orchestrator is the **sole doctrine-mediated writer**. The **agnostic core** —
the trusted act it owns on **every** harness — is **create-or-mark + provision
(ADR-006 D9 allowlist, withheld tier excluded) + marker**. The worker returns a
source delta; it never writes doctrine state.

**Per-worktree env: emission is a contract, delivery is subprocess-only.** The
binary may *compute and emit* a per-worktree env contract (`KEY=value` on stdout) —
that emission is harness-identical. **Delivery to the worker is not:** a subprocess
harness (codex/pi) injects it into the worker's environment (`env -C`/bwrap), while
**claude's `Agent` path has no worker env channel and cannot consume it** today
(IDE-004). So env emission is part of the *contract* but not part of the
*everywhere-enforced* core. The contract is **generalisable**; the **project**
declares its own consumers — doctrine-the-repo's
`CARGO_TARGET_DIR=<jail-root>/wt/<branch>` (ADR-008 D-B1) is one project-local
consumer, **never a framework primitive**.

### D3 — Per-harness capability / altitude table

The contract is uniform; the reachable altitude is not. **No harness-specific
command (`claude -p`) is a required element** — it is at most a codex/pi
enhancement layer.

| Axis | **codex / pi** (`/dispatch-subprocess`) | **claude** (`/dispatch-agent`) |
|---|---|---|
| Spawn backend | subprocess (`codex exec` / pi) binding worker cwd via `env -C "$D"` / bwrap `--chdir` | in-session **`Agent` tool**, `isolation: worktree` — **first-class, not a degraded rung** |
| Worker identity | disk marker (primary) **+** `DOCTRINE_WORKER` env (optimisation) | **disk marker only** — no env channel |
| Marker writer | `fork --worker` (orchestrator-owned verb) | **matcher-scoped `SubagentStart` hook** `marker --stamp-subagent` (Claude does default creation; hook provisions+stamps into `cwd`). WorktreeCreate `create-fork` **deferred** — payload lacks type/path/base (SL-056 PHASE-02) |
| Base pinning | **explicit `fork --base <B>`** | **opaque residual (charge-2, D5)** — Claude chooses the base; not orchestrator-controlled |
| Per-wt build isolation | per-worktree `CARGO_TARGET_DIR` (ADR-008 D-B1) | **none** — shares the jail-wide target; the §5.1 rituals stand (ADR-008) |
| OS confinement | nested bwrap (ADR-008 D-B3, **spike/D6-contingent**) | **none** — `Agent` is not a subprocess to wrap (SL-056 §4c) |
| Worker-on-main catch | **yes** (env leg) | **no** — deferred D2b residual, mitigated by always-isolating + the hook-stamped marker |
| Concurrency | file-disjoint **execution** first-class; **one landing per base** (υ) | same — file-disjoint execution first-class; one landing per base |
| Fail-closed altitude | full mechanism floor **under D6**; accident-fenced absent D6 | **O3-red, RESOLVED (D6)** — SubagentStart-stamp: blocking ⇒ stamp-before-worker on success; **not fail-closable** (read-only event); privilege-fenced by the marker-absent fail-closed rule (ADR-006 D2a) |

### D4 — codex/pi enhancements ride the subprocess seam; the marker floor does not

Three enhancements are **codex/pi-only until a free claude env backend lands**
(IDE-004 channels — optimisation tier, never required): (a) env-arm
`DOCTRINE_WORKER` (an optimisation of the marker, not the identity); (b) per-worktree
env *delivery* (the generalisable emission contract of D2 reaching the worker, of
which `CARGO_TARGET_DIR` is a project-local consumer); (c) nested bwrap (ADR-008
D-B3).

**Base pinning is not uniform.** Base-pinned `fork --base <B>` is delivered for
subprocess harnesses (codex/pi). **Claude receives provision + marker through the
matcher-scoped `SubagentStart` stamp hook (atop its own default creation), but NOT
explicit base pinning** — Claude chooses the base opaquely, and its base-pinning
residual is D5. (The `create-fork` create+provision+stamp-in-one-act path is deferred —
SL-056 PHASE-02; D6.) The agnostic core
(create-or-mark + provision + marker — D2) has live customers on all three staged
jail harnesses (`jailed-pi`, `jailed-claude`, `jailed-codex`); base-pinned fork and
the three subprocess enhancements have customers on codex/pi only.

### D5 — claude base-pinning is a confessed residual, not parity (charge-2)

**Sharper under SubagentStart-stamp (SL-056 PHASE-02/03):** Claude performs the
worktree creation and **chooses the base itself** — no orchestrator-supplied base
reaches any hook, and the spike observed the default-created worktree HEAD was **not**
the orchestrator's HEAD. So the base is **opaque and not orchestrator-controlled** —
weaker than the original "forks from session HEAD" assumption. This ADR does **not**
claim hook-time base parity with codex/pi's explicit `fork --base <B>`.

A wrong base is **usually** caught late: if the worker's distilled commit has a parent
`S^ != B`, `import` refuses **`multi-commit`** (SL-056 §7a — *not* `head-moved`, which
is the orthogonal `HEAD != B` orchestrator-side check), and the worker re-dispatches —
cost a **wasted worker run**. **But the catch is not total (M1 residual, accepted for
v1):** a delta that *presents* `S^ == B` (parent shown as `B`) while its tree was
developed against an opaque `B'` passes `multi-commit`, passes the belt, and can pass
`git apply --3way` **cleanly but semantically wrongly** (textual non-overlap ≠ semantic
correctness). So the worst case is a **clean-applying, semantically-wrong import**, not
always a bounded refusal — a confessed residual, not a closed case. An import-time
content-base assertion that would close it is **deferred to IMP-043** (owner-accepted
for v1); no hook-time `base-moved` pre-refusal is possible (no hook sees a base).

### D6 — claude fail-closed altitude (φ) — RESOLVED to O3-red (SL-056 PHASE-02/03)

**The O3 spike came back RED and PHASE-03-A settled the SubagentStart altitude; this
cell is now asserted, not proposed.** The deployed Claude Code `WorktreeCreate` payload
carries no `agent_type`/`worktree_path`/base, so the fail-closed create-replacing arm
is **not achievable** (it can neither classify, target, nor base-pin a worker); it is
**deferred** until the payload grows those fields (or an IDE-004 channel lands). The
achievable altitude is **SubagentStart-stamp**:

- A **matcher-scoped, sync-blocking** `SubagentStart` hook provisions + stamps the
  marker into the worker's `cwd`. Because the hook blocks the worker until it exits,
  **the marker is mechanically present before the worker's first command — on hook
  success.** This is **stronger than the original O3-red framing on the race axis**:
  there is *no* worker-execution race (the "created-but-unstamped window" is not a
  timing race against the worker).
- **But SubagentStart is a read-only event** (docs + PHASE-03-A: no exit status, incl.
  `exit 2`, can abort the subagent). So the residual fail-open is a **hook-failure
  case** (the stamp errors → the worker runs unstamped), **un-gateable by the hook**.
  This is **not fail-closable**.

**The fail-open is contained by the funnel + the jail, not by fail-closing the write**
(re-amended SL-056 PHASE-05, owner-locked VH; supersedes the G2 B2/B3 fail-closed
framing — see ADR-006 D2a). A stamp-failed worker runs `marker_present=false` and is
therefore **not** refused at the CLI write seam: refusing it would contradict ADR-006
D6a's solo-in-worktree direct-writer, and re-weighed against the actual risk —
`P(hook failure) ≈ 0` (the hook blocks; a miss needs a crash) **×** *jail-bounded* harm
(no push ⇒ worst case is lost local work) — the fail-closed floor bought negligible
security over the funnel. Its blast radius is bounded instead by: the **dispatch funnel
`import` belt** (`.doctrine/`+`.claude/`-rejection, SL-056 §7 — an unstamped worker
writing into its own fork is caught at import, report+halt); the **pre-distilled
prompt**; and the **bubblewrap jail with no push access** (ADR-008), which caps the
worst case at *lost unpushed progress*. The residual under this posture is not *harm*
but *silence* — an unstamped worker is signal-identical to a legit solo writer, so a
failing hook could quietly become the status quo; this is closed by an **orchestrator
post-spawn marker check that aborts the unstamped fork** (IMP-052), enforcing at the one
layer that **can** abort where this read-only hook cannot. The sole marker-minting verb
(`marker --stamp-subagent`) is exempt **by verb identity** (its `cwd` *is* a linked
worktree). With this posture, claude worker-sole-writer is **funnel-fenced +
jail-bounded + prompt-enforced + spawn-time-observable** — still **not malice-proof**
against an OS-level escape (no env-lock; bwrap is codex/pi-only) until IDE-004 /
userns-bwrap (ADR-008 D-B3; macOS analog IMP-045).

**Matcher caveat (M2):** the literal `dispatch-worker` matcher path is **doc-supported
but not yet end-to-end probed** (the agent-def registry needs a fresh session; PHASE-03-A
proved the mechanism with `general-purpose`). A matcher↔`DISPATCH_WORKER_AGENT_TYPE`
drift makes the hook **not fire at all** ⇒ unstamped worker ⇒ caught by the funnel
`import` belt + the IMP-052 orchestrator post-spawn marker abort above (not free). A
fresh-session matcher probe is owed (tracked IMP).

### D7 — σ blast-radius — WITHDRAWN (SL-056 PHASE-02/03; no longer a residual)

σ was an artifact of the create-replacing `WorktreeCreate` hook, which would have
intermediated creation for *every* worktree and so needed a shared create+provision
core and a benign else-branch (the blast radius). **The SubagentStart-stamp path does
not intermediate creation at all** — Claude creates every worktree by default; the hook
only fires per-subagent and stamps, and its **matcher scopes the stamp to
`dispatch-worker`**. There is **no else-branch and no shared creation core**, so the
blast radius does not arise. The "benign isolated subagents gain `.worktreeinclude`
provisioning" upside was a *side-effect* of creation-replacement, **never a
requirement** — the SL-056 PHASE-03 review confirmed **no skill or verb depends on a
benign isolation:worktree subagent being provisioned** (`/worktree` provisions its own
forks; `/dispatch` forbids `Agent isolation:worktree`). Benign isolated subagents now
revert to Claude's bare default creation (unprovisioned) — **a deliberate scope cut with
zero dependents, not a functional loss.** This confession is withdrawn; the matcher
question and the delete-vs-keep-else-branch disposition both dissolve.

## Consequences

### Positive

- Each mechanism moved into a verb makes the worktree/dispatch skills **shorter and
  more harness-agnostic in one move** — the token payoff and the agnosticism payoff
  are the same edit.
- The agnostic floor (marker + provision + per-wt env emission) is **identical and
  golden-testable** under claude/codex/pi; trust no longer rides prose an LLM may skip.
- A single contract admits **per-harness capability layering** without a uniform
  altitude lie — the table states honestly what each harness can and cannot enforce.

### Negative

- The claude fail-closed altitude **settled at the O3-red SubagentStart-stamp row**
  (SL-056 PHASE-02/03): **not fail-closable** (read-only event), contained instead by
  the marker-absent fail-closed privilege rule (ADR-006 D2a). Residuals (opaque base
  D5/M1, worker-on-main, OS-level malice) are real and named, not closed.
- The literal `dispatch-worker` **matcher path is doc-supported but not yet
  end-to-end probed** (fresh-session registry; tracked IMP) — a matcher drift fails
  into the marker-absent fail-closed rule, not into a free-writer.

### Neutral

- The subprocess enhancements (env-arm, per-wt env, bwrap) are optimisation-tier and
  **codex/pi-only** until a free claude env backend (IDE-004) lands.
- This ADR is a **contract, not an amendment**: ADR-006's D2a/D5/D9 text changes are
  the separate, post-spike SL-056 G2 step.

## Verification

- VA — the contract reads agnostically: the everywhere-enforced core (create-or-mark
  + provision + marker) is stated as harness-identical; per-wt env *emission* is a
  contract whose *delivery* is subprocess-only; and base-pinning, `DOCTRINE_WORKER`,
  bwrap, and the per-wt target are stated as codex/pi-only, never the identity.
- VA — the altitude table shows codex/pi vs claude on each axis; no harness-specific
  command (`claude -p`) appears as a required element.
- The **φ claude fail-closed cell is RESOLVED to the O3-red SubagentStart-stamp row**
  (D6): blocking ⇒ stamp-before-worker on success, but not fail-closable; the
  marker-absent fail-closed privilege rule (ADR-006 D2a) is what contains the
  stamp-failure case, not the hook.
- The **σ blast radius is WITHDRAWN** (D7): SubagentStart does not intermediate
  creation, so there is no shared-core else-branch; benign isolated subagents revert to
  Claude default creation (no dependents — a scope cut, not a loss).
- The **charge-2 base-pinning residual** is named as a residual, not parity, and is
  **sharper** under Claude-chosen opaque bases (D5/M1).
