# IMP-031: lightweight patch or update artifact

There's more wrapping than sandwich sometimes using slice > design > plan >
phase > audit for small changes.

Consider either a "patch mode" for tiny changes, or a non-slice (patch? card?)
primitive for very simple changes.

Need to be very careful not to bypass planning where it's appropriate, or to 
introduce a vector for breakage / bad code without typical safeguards.
