# IMP-164: Seed-onboarding template and project orientation are empty stubs

## Context

`install/templates/seed-onboarding.md` (the template seeded into client projects
as `mem.signpost.project.orientation`) was a completely empty stub — all 6
sections were `<!-- Describe ... -->` HTML comment placeholders. The local
`mem.signpost.project.orientation` for this repo was identical.

The boot digest directs every agent to retrieve this memory on their first turn.
A dead-end retrieval wastes context and leaves the agent with no project
orientation.

## Done

1. **Seed template rewritten** (`install/templates/seed-onboarding.md`):
   action-oriented, agent-audience prompts. POL-002 compliant — no
   doctrine-specific conventions baked in. Same 6-section structure preserved.

2. **This project's orientation filled** (`mem.signpost.project.orientation`):
   concrete content covering purpose, principles, architecture, structure,
   conventions, tooling, and further reading.

## Remaining

- **Memory verification blocked** — working tree has unrelated untracked files
  from other work. `doctrine memory verify mem.signpost.project.orientation` is
  needed after cleanup.
- **install/governance.md** is also an empty stub ("Where things live" and
  "Project-local rules" sections blank). Separate follow-up.
- **`.doctrine/templates/seed-onboarding.md`** — this copy is gitignored and
  regenerated by `doctrine install`; the source of truth is
  `install/templates/seed-onboarding.md`.
