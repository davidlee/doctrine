# RFC-004: Path-intent selector: role-stamped selector+intent primitive

> Status: **open** — prove-value stage. Combines design, governance, and research.
> Asserts no canon. Goal of this RFC is to decide *whether* a primitive is worth
> extracting, not to lock its shape.

## Context

Several Doctrine surfaces independently reinvent the same shape: **a set of
files/globs, an intent for why they matter, and a provenance (who/what stage
stamped them).** Today the most visible instance — the review `domain_map`
(ADR-007 §D-C10) — is, in practice, a token sink: agents hand-author it, fail
validation across several rewrites, eventually copy a known-good example, and
**nothing reads it back**. The warm-cache consumer it was built for is unproven.

The naïve fix (CLI + reference memory + MCP support to cheapen authoring) is a
trap: it makes *production* cheaper without creating a *consumer*. A cheaper way
to produce something nobody reads is still zero value.

But the diagnosis points somewhere more interesting. The `domain_map` is dead
because it is authored **once, in isolation, by the wrong actor at the wrong
time** (the reviewer, cold, at review start). The same data, made to **accrete
through the lifecycle**, gives each stage both a reason to touch it and a payload
to add — and surfaces a real, mechanical consumer.

## Discussion

### Decompose the bundle

`domain_map` bundles two things with opposite economics:

- **Prose mental model** (areas/purpose, invariants, risks). Agent-authored,
  agent-consumed. This is the part that is never read. Tooling cannot rescue it.
- **Tracked path-set + hashes.** Mechanical. `review status` → `stale_paths`.
  Real signal, no prose required. Needs only the path list, not the model.

The bundle is the bug. We pay a validation tax to produce the dead prose tier
just to carry the cheap mechanical tier.

### The accretion model surfaces the real consumer

If the path-set accretes by stage:

- **slice author** seeds `intent = scope-relevant` (files in play).
- **designer** extends with `intent = design-target` (create / modify), so they
  don't start cold finding+annotating the same files.
- **orchestrator** records `intent = actual-delta` from the git-changed manifest
  during patch application (check + record in one).
- **auditor** reads the **differential**: declared design-targets vs actual
  delta. The gap — files touched but undeclared, declared but untouched, deleted
  unmentioned — is exactly the conformance signal the audit hunts by hand today.

**That diff is the killer consumer.** It is mechanical, high-value, needs no
prose, and retroactively justifies the structure — unlike warm-cache survival.

Critically: accretion justifies **path-set + intent only**. It does *not*
re-justify the invariants/risks prose. Those stay optional/separate/earned.

### The primitive

```
target     : EntityRef | InternalDocRef | PathRef(path|glob) | ExternalRef(url)
                                 # what — selectors use the non-entity arms (OQ-6)
intent     : enum                # why — scope-relevant | design-target(create|modify)
                                 #       | actual-delta(created|modified|deleted)
layer      : L0 | L1 | L2        # read-relevant | declared | actual (OQ-8)
provenance : stage (+ role, PHASE-NN) + when   # at what authority it was stamped
prose?     : annotation          # optional, durable ONLY when anchored here (OQ-7)
```

A slice carries a *list* of these, accreting by stage. Staleness keys off
resolving the selector and hashing — falls out free.

### Candidate consumers (the primitive is a unification, not a feature)

1. **review `domain_map`** — current instance; path-set tier survives, prose tier deprecated.
2. **audit conformance** — declared-vs-actual path diff (the killer consumer).
3. **dispatch disjointness** — file-disjoint phase parallelization is a path-set
   disjointness check; if phases declared selectors, it's mechanical not eyeballed.
4. **slice scope** — "files in play" is just `intent = scope-relevant`.
5. **staleness** — `review status` already consumes the path-set.
6. **IMP-012 backlog triggers** — the backlog schema *already* carries
   `triggers = []  # watched source globs: { globs, note? }`. That is
   `selector(globs) + intent(note) + provenance(the item)` — the same primitive,
   sense-checked at a planning gate. A structural twin, not a cousin.
7. **All RV-format surfaces** (future) — audit-RV is consumer #2, but
   **inquisition** and **code-review** (IMP-042, being dragged tighter into the
   core loop; already on the RV format via IMP-023) ride the same ledger. Every
   RV-producing surface wants the same two things this RFC mints: a context
   cheatsheet (the map, as a warm start instead of cold re-exploration) **and**
   an accurate file-delta (the recorded-SHA union, OQ-11). So the consumer is not
   "the auditor" specifically — it is *any review that opens against a slice's
   changes*. This is the generalisation of consumer #2, not a separate one.

So this is not "build a new feature." It is "name a primitive that ≥6 surfaces
are already reinventing, then earn each consumer."

### A selector is a non-entity edge target

The relation engine today links **entity → entity** (ADR-004 outbound-only,
ADR-010 unified contract / bespoke storage). Every target has an id, a `show`, a
derived reverse edge. A selector breaks that assumption deliberately: its target
is **not an entity**. It points at things worth pointing at that have no entity
record —

- raw repo paths / globs (code, configs);
- an entity's *internal* markdown that is itself not an entity (a slice's
  `design.md` is a tier of `SL-NNN`, not a `DOC-NNN`);
- external refs (a URL, an upstream doc).

This is a **new edge-target class**: *terminal* (no reciprocity — a file can't
point back), *resolvable* (path/glob → concrete file set), *hashable* (the
staleness key). The clean framing under ADR-010: unify the **contract** — an edge
is `(label, target)` — and let `target` become a sum type
`{ EntityRef | InternalDocRef | PathRef(path|glob) | ExternalRef(url) }`, keeping
storage bespoke per target class. That is ADR-010's own philosophy, not a
departure from it. Whether this lives *inside* the relation engine or *beside* it
is **OQ-6** — forcing non-entity targets through the entity-id relation table
would pollute the "every target resolves via `show`" invariant.

### The map as a progressive planning artifact (prose, anchored)

A variation of this map is the **structured output of a research/planning stage**
— the thing preflight and design produce instead of dying prose. It refines by
stage, gaining resolution as confidence grows:

- **scoping pass** (preflight / `slice`): coarse, broad globs, `intent =
  scope-relevant`. "These areas are in play." Low confidence.
- **detailed pass** (`design` / `plan`): fine, exact paths, per-`PHASE-NN`,
  `intent = design-target`. "This phase creates X, modifies Y."

This gives research agents a **machine-checkable output contract** in place of
free prose — and prose findings die exactly the way `domain_map` prose died
(OQ-5). The discipline that falls out, and the principled rescue of the prose
problem: **prose survives iff it is anchored to a selector.** An invariant or
risk hanging off a concrete target (file/area) is durable because the target
gives the next stage a reason to resolve it; an *unanchored* invariant is the
dead tier. So we don't ban prose — we require it to hang off a `PathRef`. This is
**OQ-7**: is unanchored prose forbidden, or merely flagged non-durable?

### The auditor's map is layered, not flat

The map an auditor wants is a **stack of layers**, keyed by intent + provenance +
stage — not one file list:

- **L0 read-relevant** — informed the design, read but not (meant to be) changed.
  Provenance: slice author / designer.
- **L1 declared-target** — will create / modify / delete. Provenance: designer /
  planner, attributed per `PHASE-NN`.
- **L2 actual-delta** — git says created / modified / deleted. Provenance:
  orchestrator / git, *derived not stored* (OQ-1).

The audit product is the **cross-layer set algebra**, and each cell is a distinct
signal:

- `L1 \ L2` — declared but undelivered (claimed, never touched).
- `L2 \ L1` — undeclared change (scope creep / collateral edit).
- `L2 \ (L0 ∪ L1)` — total surprise: touched something never mentioned at any
  stage. Highest audit signal.
- `L0 ∩ L2` — read-then-changed: was a target-declaration missed?
- `L1 ∩ L2` — conformant.

This is "lots to poke at" made concrete. It also pressures the data model: an
entry needs intent **and** layer **and** per-phase attribution — watch OQ-4 (the
layer vocabulary wants to stay small, **OQ-8**: fixed `{L0,L1,L2}` enum vs open
intent vocabulary).

### Phase criteria, VTs, and what git can verify

Phases carry immutable EN / EX / VT criteria. Git knows the exact changed-file
set. The tie-in: a phase's **L1 declared-target ∩ git L2 actual-delta** is a
mechanically checkable EX criterion — "phase exits when it touched what it said
it would, and nothing it didn't."

But pin the limit precisely: **path-conformance is necessary, not sufficient.**
Changing the right files ≠ changing them correctly. It catches *undeclared* edits
and *undelivered* declarations; it cannot judge whether a change is right. So it
is a **conformance gate, not a correctness gate** — it tells the auditor *where
to look*, not *whether it passes*. That argues against minting a new verify mode:
the git-delta is **evidence consumed by an existing VA/VH**, not a new VG axis
(**OQ-9**). Progressive refinement raises **OQ-10**: when planning paths supersede
scoping globs, do both layers persist (provenance trail) or does the coarse one
get replaced?

### Tensions to settle (open questions)

- **OQ-1 Storage tier split.** Intent declarations (slice scope, design targets)
  are *authored* — committed, diffable. Actual-delta is *derived* — git already
  knows it; storing it risks drift against real git. The audit diff is
  *computed*, not stored. The primitive likely spans tiers. Don't persist what
  git gives free.
- **OQ-2 Glob intent vs glob resolution.** A glob is intent-stable but resolves
  to a drifting set. Staleness wants the *resolution* hashed; intent wants the
  *glob* kept. Probably store both — glob is durable identity, resolution is the
  drift key.
- **OQ-3 Axis: role or stage?** The same agent plays many roles. The auditor
  cares *at what stage / what authority* a claim was stamped, not *who*. Lean
  provenance = stage primary, role secondary.
- **OQ-4 Premature generalization.** ≥6 candidate readers is suggestive, not a
  mandate. Model the data shape once; wire consumers as each is *proven*. Don't
  ship six integrations on faith — that repeats the `domain_map` mistake one
  level up.
- **OQ-6 Edge-target taxonomy & home.** Does the non-entity target sum type live
  *inside* the relation engine (risking the "every target has a `show`/reverse"
  invariant) or *beside* it as a parallel terminal-link store? ADR-010 says unify
  the contract, keep storage bespoke — apply it here.
- **OQ-7 Prose anchoring.** Is unanchored prose (an invariant/risk with no target)
  forbidden, or merely flagged non-durable? The rescue of the dead prose tier
  hinges on this rule.
- **OQ-8 Layer vocabulary.** Fixed `{L0 read-relevant, L1 declared, L2 actual}`
  enum, or an open intent vocabulary? Small-and-fixed resists model sprawl; open
  is more expressive. Lean fixed until a consumer needs more.
- **OQ-9 Verification mode.** Is path-conformance a new verify mode (VG, by git
  delta) or merely *evidence* feeding an existing VA/VH? Lean evidence — it is a
  conformance gate, not a correctness gate, so it cannot stand alone as a verdict.
- **OQ-10 Refinement persistence.** When detailed-pass paths supersede
  scoping-pass globs, do both layers persist (a provenance trail of how scope
  narrowed) or is the coarse layer replaced? Audit may want the trail.
- **OQ-5 Is the prose tier dead, or just weak?** *Settled — dead.* Call-graph
  audit of `src/review.rs` + `src/mcp_server/tools.rs`: the sole writer is
  `write_cache` (review.rs:2504) via `run_prime`. The prose tier
  (`area.purpose`, `invariant.text`, `risk.text`) has **zero runtime readers** —
  structurally typed, never inspected; `validate_domain_map` only checks
  `is_empty`. The path-set tier has exactly **one** weak reader: `run_status`
  (review.rs:2710) → `tracked_paths()`/`baseline()` → `stale_paths`, exposed via
  CLI `review status` and MCP `review_status`. `review_show` reads the authored
  brief, not the cache. Conclusion: deprecating the prose tier breaks nothing;
  the path-set tier is the only thing worth keeping, and even it is barely used.

### Approach to proving value

Earn it through the killer consumer first: prototype the audit declared-vs-actual
diff against one real slice. If that diff produces conformance signal an auditor
would otherwise find by hand, the primitive is justified and the other consumers
follow. If it doesn't, strip `domain_map` to an auto-seeded path-set and stop.

## v0.1 — constrained-but-sufficient scope

North star: **at audit, conformance drift between what the design declared it
would touch and what git shows it touched is computed, not hand-hunted.** v0.1 is
the smallest thing that delivers that for four roles — slice author, design
author, reviewer, auditor — and nothing else.

### The cut that simplifies everything

**The L2 actual-delta *file-set* is never stored — git is the source.** Only
declarations are authored; the actual file-set and the diff are computed on demand
at audit.

*Clarification (supersedes the original "no orchestrator writer" line; resolved
later in this RFC by OQ-11a).* "Never stored" governs the L2 **file-set**, not the
commit identity needed to compute it. OQ-11a (below) establishes that the platform
**does** record each phase's **source-delta SHA** as runtime/derived state at
land-time — a minimal recording beat, not the dead L2 file-set. So v0.1 has no
writer *of actual-delta file-sets*, but it does carry a thin SHA-boundary writer
on each landing arm. The two are consistent: SHAs are the owned input; the file-set
is still derived on demand. The dispatch funnel captures its SHA automatically;
the **solo arm has no automatic beat and so depends on an explicit SHA-recording
call** — absent it, conformance degrades to manual audit (no fabricated
substitute). (SL-147 design D5/F-1 reconciled this contradiction.)

### v0.1 data shape (minimal)

One authored, committed list per slice. Each entry:

```
path     : path | glob          # PathRef only — target sum type deferred
intent   : scope-relevant | design-target   # = L0 / L1; L2 is computed, not declared
note?    : one line             # optional, anchored; NOT the old prose tier
```

- **No per-`PHASE-NN` attribution** — slice-level declaration only. (Per-phase is
  the obvious v0.2; the slice-level diff already delivers the north star.)
- **No create/modify/delete verb from the author** — they declare *intent to
  touch*; git supplies the verb. Don't make authors predict what git observes.
- **`layer` collapses into `intent`** in v0.1 — no separate enum.

### Roles & flow

- **slice author** seeds coarse `scope-relevant` globs ("touches the review
  subsystem"). Warm-starts the designer; gives a coarse scope-creep frame.
- **design author** refines to specific `design-target` paths. *These are the
  entries the audit diff keys on* — the load-bearing input.
- **reviewer** is the consumer that *already exists* — `review status` →
  `stale_paths` (review.rs:2710). v0.1 re-points it at the unified declared list
  instead of a hand-authored `domain_map`, so the existing staleness reader keeps
  working while the dead prose authoring is deleted. Net: the reviewer gets the
  same signal for zero new authoring tax. Highest-leverage role in v0.1 — it
  proves the migration on the one path with a live reader.
- **auditor** runs one new consumer: resolve `design-target` globs → set, take git
  name-status over the slice's change range → set, emit the algebra:
  `declared ∩ actual` (conformant) · `declared \ actual` (undelivered) ·
  `actual \ declared` (undeclared / surprise). Reuses the existing
  resolve+hash seam (`tracked_paths`/`baseline`, review.rs).

### Explicitly deferred (goal in back of mind, not in v0.1)

target sum type / non-entity edge generalization (OQ-6) · per-PHASE attribution ·
verb sub-tags · new verify mode (OQ-9) · prose invariants/risks · review
`domain_map` migration · dispatch disjointness · IMP-012 wiring · MCP surface
(a thin reader is plausible, but CLI-first).

### The one new open question v0.1 forces

- **OQ-11 Slice-delta definition.** *Resolved — union of own commits, not an
  endpoint range.* Endpoint diffing is rejected: a slice spans phases and merges
  interleaved trunk between them (the edge/main promotion dance), so `base..head`
  balloons to "everything in the repo since start," and three-dot `base...head`
  *loses* slice work before the last merge as the merge-base advances. Endpoint
  diffs cannot separate "mine" from "merged-in" once a branch absorbs trunk.
  Instead: **actual-delta = ⋃ `name-status` of the slice's own source-delta
  commits**, each `sha^..sha` (or `--first-parent`); merge commits that pull
  `main` in are excluded, so interleaved trunk contributes nothing. No base ref
  needed. This leans on one existing platform invariant: dispatch lands one
  source-delta commit per phase (ADR-006 / ADR-012).
  - **OQ-11a Commit identification.** *Resolved — recorded SHAs, no grep arm.*
    `(SL-NNN)` commit scoping is a **host-project convention** (this repo's
    AGENTS.md), not a doctrine-as-platform contract — projects adopting doctrine
    may use any commit style, so a conformance mechanism that greps for it is a
    layering violation. The platform records each phase's source-delta SHA as
    runtime/derived state at land-time, made foolproof by the dispatch funnel
    (which already emits exactly one source-delta commit per phase — capture its
    sha at that beat) and the solo `/execute` path. Squash-durability falls out:
    SHAs are captured on the slice/coordination branch and ADR-012 preserves
    those code branches, so audit computes from the preserved commits regardless
    of how `main` later integrates (squash / ff / merge).

### Design principle surfaced

**Platform mechanisms must not depend on host-project conventions.** Anything
doctrine *enforces or computes* (here, conformance) rides on contracts doctrine
itself owns (recorded SHAs, the dispatch funnel) — never on a host's commit
style, branch names, or directory habits. Conventions can *inform* (a nicer
default) but never *load-bear*. Generalises beyond this RFC — **promoted to
POL-002** (required), unifying it with the prior product-not-compromised facet.

## Outcome

Open. OQ-5 settled (prose tier confirmed dead, path-set tier single weak
reader). Remaining decision: prototype the audit declared-vs-actual diff against
one real slice to prove the killer consumer before extracting the primitive. If
that signal lands, the primitive is justified; if not, the fallback is to strip
`domain_map` to an auto-seeded path-set and stop.
