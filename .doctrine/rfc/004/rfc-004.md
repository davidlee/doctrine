# RFC-004: Path-intent selector: role-stamped selector+intent primitive

> Status: **open** — prove-value stage. Combines design, governance, and research.
> Asserts no canon. Goal of this RFC is to decide *whether* a primitive is worth
> extracting, not to lock its shape.

## Context

Several Doctrine surfaces independently reinvent the same shape: **a set of
files/globs, an intent for why they matter, and a provenance (who/what stage
stamped them).** Today the most visible instance — the review `domain_map`
(ADR-007 §D-C10) — is, in practice, a token sink: agents hand-author it, fail
validation across several rewrites, eventually copy a known-good example, and
**nothing reads it back**. The warm-cache consumer it was built for is unproven.

The naïve fix (CLI + reference memory + MCP support to cheapen authoring) is a
trap: it makes *production* cheaper without creating a *consumer*. A cheaper way
to produce something nobody reads is still zero value.

But the diagnosis points somewhere more interesting. The `domain_map` is dead
because it is authored **once, in isolation, by the wrong actor at the wrong
time** (the reviewer, cold, at review start). The same data, made to **accrete
through the lifecycle**, gives each stage both a reason to touch it and a payload
to add — and surfaces a real, mechanical consumer.

## Discussion

### Decompose the bundle

`domain_map` bundles two things with opposite economics:

- **Prose mental model** (areas/purpose, invariants, risks). Agent-authored,
  agent-consumed. This is the part that is never read. Tooling cannot rescue it.
- **Tracked path-set + hashes.** Mechanical. `review status` → `stale_paths`.
  Real signal, no prose required. Needs only the path list, not the model.

The bundle is the bug. We pay a validation tax to produce the dead prose tier
just to carry the cheap mechanical tier.

### The accretion model surfaces the real consumer

If the path-set accretes by stage:

- **slice author** seeds `intent = scope-relevant` (files in play).
- **designer** extends with `intent = design-target` (create / modify), so they
  don't start cold finding+annotating the same files.
- **orchestrator** records `intent = actual-delta` from the git-changed manifest
  during patch application (check + record in one).
- **auditor** reads the **differential**: declared design-targets vs actual
  delta. The gap — files touched but undeclared, declared but untouched, deleted
  unmentioned — is exactly the conformance signal the audit hunts by hand today.

**That diff is the killer consumer.** It is mechanical, high-value, needs no
prose, and retroactively justifies the structure — unlike warm-cache survival.

Critically: accretion justifies **path-set + intent only**. It does *not*
re-justify the invariants/risks prose. Those stay optional/separate/earned.

### The primitive

```
selector   : [paths | globs]     # what
intent     : enum                # why — scope-relevant | design-target(create|modify)
                                 #       | actual-delta(created|modified|deleted)
provenance : stage (+ role) + when   # at what authority it was stamped
```

A slice carries a *list* of these, accreting by stage. Staleness keys off
resolving the selector and hashing — falls out free.

### Candidate consumers (the primitive is a unification, not a feature)

1. **review `domain_map`** — current instance; path-set tier survives, prose tier deprecated.
2. **audit conformance** — declared-vs-actual path diff (the killer consumer).
3. **dispatch disjointness** — file-disjoint phase parallelization is a path-set
   disjointness check; if phases declared selectors, it's mechanical not eyeballed.
4. **slice scope** — "files in play" is just `intent = scope-relevant`.
5. **staleness** — `review status` already consumes the path-set.
6. **IMP-012 backlog triggers** — the backlog schema *already* carries
   `triggers = []  # watched source globs: { globs, note? }`. That is
   `selector(globs) + intent(note) + provenance(the item)` — the same primitive,
   sense-checked at a planning gate. A structural twin, not a cousin.

So this is not "build a new feature." It is "name a primitive that ≥6 surfaces
are already reinventing, then earn each consumer."

### Tensions to settle (open questions)

- **OQ-1 Storage tier split.** Intent declarations (slice scope, design targets)
  are *authored* — committed, diffable. Actual-delta is *derived* — git already
  knows it; storing it risks drift against real git. The audit diff is
  *computed*, not stored. The primitive likely spans tiers. Don't persist what
  git gives free.
- **OQ-2 Glob intent vs glob resolution.** A glob is intent-stable but resolves
  to a drifting set. Staleness wants the *resolution* hashed; intent wants the
  *glob* kept. Probably store both — glob is durable identity, resolution is the
  drift key.
- **OQ-3 Axis: role or stage?** The same agent plays many roles. The auditor
  cares *at what stage / what authority* a claim was stamped, not *who*. Lean
  provenance = stage primary, role secondary.
- **OQ-4 Premature generalization.** ≥6 candidate readers is suggestive, not a
  mandate. Model the data shape once; wire consumers as each is *proven*. Don't
  ship six integrations on faith — that repeats the `domain_map` mistake one
  level up.
- **OQ-5 Is the prose tier dead, or just weak?** *Settled — dead.* Call-graph
  audit of `src/review.rs` + `src/mcp_server/tools.rs`: the sole writer is
  `write_cache` (review.rs:2504) via `run_prime`. The prose tier
  (`area.purpose`, `invariant.text`, `risk.text`) has **zero runtime readers** —
  structurally typed, never inspected; `validate_domain_map` only checks
  `is_empty`. The path-set tier has exactly **one** weak reader: `run_status`
  (review.rs:2710) → `tracked_paths()`/`baseline()` → `stale_paths`, exposed via
  CLI `review status` and MCP `review_status`. `review_show` reads the authored
  brief, not the cache. Conclusion: deprecating the prose tier breaks nothing;
  the path-set tier is the only thing worth keeping, and even it is barely used.

### Approach to proving value

Earn it through the killer consumer first: prototype the audit declared-vs-actual
diff against one real slice. If that diff produces conformance signal an auditor
would otherwise find by hand, the primitive is justified and the other consumers
follow. If it doesn't, strip `domain_map` to an auto-seeded path-set and stop.

## Outcome

Open. OQ-5 settled (prose tier confirmed dead, path-set tier single weak
reader). Remaining decision: prototype the audit declared-vs-actual diff against
one real slice to prove the killer consumer before extracting the primitive. If
that signal lands, the primitive is justified; if not, the fallback is to strip
`domain_map` to an auto-seeded path-set and stop.
