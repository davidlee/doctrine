## 1. Design Problem

`cargo install doctrine` requires a local Rust compile, which fails on macOS
toolchains that can't resolve `iconv` at link (`ld: library not found for
-liconv`) and is fragile across host environments generally. Provide an install
path that needs **no local compile**, so any macOS user can install regardless
of their toolchain state.

## 2. Current State

- Only delivery path is `cargo install` / source build → local compile.
- No CI exists (`.github/` absent); release is local `just`:
  `just release <bump>` bumps + gates (`gate nix-build`) + commits + tags;
  `just publish` runs `cargo publish --allow-dirty` (crates.io).
- The binary is **not** standalone — it embeds assets via `rust-embed`
  (`install/`, `plugins/`, `memory/`) and `include_str!`
  (`.pi/extensions/doctrine/mcp.ts`), plus the RustEmbed map over
  `web/map/dist/` (generated by `just web-build`, gitignored). A git-based source
  listing drops `dist` ([[crane-strips-non-rust-embeds]]); the nix flake (crane)
  grafts it back. A plain `cargo build` ships a broken map unless `web-build` runs
  first.

## 3. Forces & Constraints

- **Env independence** — must work on a macOS box with a broken/odd toolchain;
  rules out "build locally and upload" (the operator's own Mac is the broken
  case) and any approach that compiles on the user's machine.
- **Embed integrity** — the shipped artifact must carry every embed root; this is
  the dominant correctness risk and must be gated on the *artifact*, not on
  `cargo test`.
- **Minimal new surface** — the repo is deliberately nix + `just`; introducing CI
  is justified only because it is the sole env-independent producer of macOS
  binaries.
- **POL-002** (platform independence from host conventions): the installer and
  workflow must not assume host-project layout; they operate on this repo's own
  release artifacts only.

## 4. Guiding Principles

- The single self-contained binary stays the unit of distribution (consistent
  with SPEC-009's "no second asset bundle, no network fetch" at install time).
- Prove embed integrity on the real artifact before it reaches a user.
- Keep `cargo install` as a fallback; prebuilt becomes the recommended default.
- DRY the smoke test: one script, run by `just smoke` locally and by CI.

## 5. Proposed Design

### 5.1 System Model

```
 git tag vX.Y.Z ──push──▶ .github/workflows/release.yml  (macos-14, arm64)
   ├─ setup: rust + targets(aarch64,x86_64-apple-darwin), bun
   ├─ just web-build                         # generate web/map/dist
   ├─ cargo build --release --target <triple>   # per arch
   ├─ scripts/smoke.sh <binary>              # embed-integrity gate
   ├─ tar.gz + sha256                        # per arch
   └─ softprops/action-gh-release            # upload assets to Release vX.Y.Z
                                   │
        ┌──────────────────────────┼───────────────────────────┐
   curl install.sh             cargo binstall              (cargo install
   (no toolchain)              (no compile)                 — fallback)
```

### 5.2 Interfaces & Contracts

**Release assets** (per tag `vX.Y.Z`), names are the contract three consumers
depend on (installer, binstall `pkg-url`, manual download):
- `doctrine-aarch64-apple-darwin.tar.gz` + `.sha256`
- `doctrine-x86_64-apple-darwin.tar.gz` + `.sha256`
- each tarball contains a single executable named `doctrine`.

**`install.sh`** (repo root; consumed as
`curl -fsSL https://raw.githubusercontent.com/davidlee/doctrine/main/install.sh | sh`):
- POSIX `sh`, `set -eu`.
- Refuse non-Darwin with a clear message (Linux is a documented follow-up).
- `uname -m`: `arm64`→`aarch64-apple-darwin`, `x86_64`→`x86_64-apple-darwin`.
- Version: `${DOCTRINE_VERSION:-<latest>}` (latest via GitHub releases API).
- Download asset + `.sha256`; verify with `shasum -a 256 -c`.
- `xattr -d com.apple.quarantine <bin> 2>/dev/null || true` (strip Gatekeeper).
- `chmod +x`; install to `${DOCTRINE_BIN_DIR:-$HOME/.local/bin}`; print PATH hint
  if the dir is not on `$PATH`.

**`[package.metadata.binstall]`** in `Cargo.toml` (read from the crates.io crate):
```toml
[package.metadata.binstall]
pkg-url = "{ repo }/releases/download/v{ version }/doctrine-{ target }.tar.gz"
pkg-fmt = "tgz"
bin-dir = "doctrine"
```

**`scripts/smoke.sh <binary>`** — exit non-zero unless ALL hold:
- `<binary> --version` succeeds.
- `<binary> install --path <tmp>` lays down a known template file (proves the
  `install/` embed). Exact file chosen at plan time from `install/templates/`.
- the embedded map serves: boot `<binary> map serve` on an ephemeral port, GET
  `/` → HTTP 200 with non-empty body, shut it down (proves `web/map/dist` graft).

### 5.3 Data, State & Ownership

- The workflow owns artifact production and naming; the GH Release is the
  durable artifact store.
- `install.sh` and `[metadata.binstall]` are *consumers* of the naming contract;
  a rename to asset names is a breaking change across all three — change in one
  place (workflow) and update the two consumers in the same commit.
- No new runtime/authored state in `.doctrine/`.

### 5.4 Lifecycle, Operations & Dynamics

- Release flow unchanged locally (`just release`), with one added step:
  `git push --tags` now triggers `release.yml`. Document this in the `justfile`
  release recipe comment.
- Versioning: the workflow derives `X.Y.Z` from the pushed tag; no version is
  hand-passed.
- Failure mode: a smoke-test failure fails the workflow and *no* assets are
  published — a broken-embed binary never reaches users.

### 5.5 Invariants, Assumptions & Edge Cases

- INV: published assets always pass `scripts/smoke.sh` (gate precedes upload).
- INV: asset names match the contract in §5.2 exactly.
- ASM: GitHub-hosted `macos-14` runners have a clean Apple SDK (no iconv issue)
  and Rosetta available to run the x86_64 smoke test on the arm64 host.
- EDGE: x86_64 is cross-compiled on the arm64 runner; if cross-link surprises,
  fallback is a separate `macos-13` (x86_64) matrix leg — same steps, native.
- EDGE: `install.sh` on an unsupported arch/OS exits with guidance, never a
  partial install.
- EDGE: quarantine strip is best-effort (`|| true`) — absence of `xattr` or the
  attr is non-fatal.

## 6. Open Questions & Unknowns

- OQ-1 Exact smoke-test assertions (which `install/templates/` file; map route +
  port handling). Resolved at plan/phase time.
- OQ-2 x86_64-on-arm cross-compile viability vs a second native runner leg —
  prove early in the first phase.
- OQ-3 SPEC-009 reconcile: binary *delivery* (download channel, installer) is new
  evergreen surface beyond SPEC-009's "embed + lay-down into a project" scope.
  Route at reconcile (extend SPEC-009 or record a boundary). Not a slice blocker.

## 7. Decisions, Rationale & Alternatives

- D1 **GitHub Actions, macOS runner** as producer. Alt: manual local mac build
  (rejected — operator's Mac is the broken case; not reproducible). Alt: nix on
  macOS CI (rejected as primary — slow cold cache, adds nix CI dep; kept as
  fallback for embed integrity if cargo+web-build proves fiddly).
- D2 **cargo + bun web-build** as the build step, gated by an **artifact smoke
  test**. Alt: `nix build .#doctrine` (stronger hermetic embed guarantee but
  heavier); the smoke test buys the embed guarantee more cheaply and tests the
  actually-shipped bytes.
- D3 **`curl|sh` installer (primary) + `cargo-binstall` metadata (complement)**;
  Homebrew tap **deferred** (real value, but a second repo + formula automation =
  scope creep). `cargo install` retained as fallback.
- D4 **macOS-only** targets now; Linux a follow-up (matrix leg makes it cheap).
- D5 **Unsigned + installer strips quarantine**; notarization deferred (needs paid
  Apple Developer account + CI secrets; revisit only if Gatekeeper friction is
  real for the non-installer download paths).

## 8. Risks & Mitigations

- R1 **Broken embed in shipped binary** (highest). Mitigation: `scripts/smoke.sh`
  gate on the artifact, exercising the `install/` embed and the `web/map/dist`
  map before any upload.
- R2 **Asset-name drift** across workflow/installer/binstall. Mitigation: single
  documented naming contract (§5.2); all three edited together.
- R3 **Cross-compile/Rosetta smoke fragility** for x86_64. Mitigation: prove in
  phase 1; documented fallback to a native `macos-13` matrix leg.
- R4 **Unsigned-binary Gatekeeper friction** on direct/binstall downloads (no
  quarantine strip there). Mitigation: document the one-time override; notarization
  available as a follow-up if it bites.
- R5 **Introducing CI to a CI-less repo** — maintenance surface. Mitigation: one
  narrow tag-triggered workflow; reuse `scripts/smoke.sh` so logic isn't
  CI-only.

## 9. Quality Engineering & Validation

- Primary gate: `scripts/smoke.sh` runs in CI on both arches (x86_64 via Rosetta)
  and locally via `just smoke` — embed integrity proven on the real artifact.
- Closure evidence: from a fresh macOS environment with no Rust toolchain, the
  `curl|sh` one-liner installs a runnable `doctrine`; `doctrine map serve` serves
  the embedded UI; `doctrine install` lays down templates.
- `install.sh` triple mapping is the fragile unit; it is exercised by the CI
  install-from-asset on both arches (behavioural, not a separate unit test).
