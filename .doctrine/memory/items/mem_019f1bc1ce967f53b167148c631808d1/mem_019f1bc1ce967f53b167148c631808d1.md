# Locate coord-owned dispatch files by layout-strip, not git-common-dir

To find a file that a dispatch worker's hook must resolve relative to the
**coordination-tree root** (e.g. the per-worktree jail policy at
`<coord>/.doctrine/state/dispatch/jail/<name>.toml`), recover that root by
**stripping the doctrine-owned `.worktrees/<name>` layout** from the worker's
cwd — NOT via git topology.

**Why git-common-dir fails.** A dispatch worker runs in a linked worktree at
`<coord>/.worktrees/<name>`. Its git-common-dir resolves to the **primary
repo's `.git`** (worktrees created off a coord tree still share the primary
object store), so `parent(common_git_dir)` is the *primary repo root*, NOT the
coord root where `.worktrees/` and the runtime `dispatch/` state actually live.
Using git-common-dir to locate coord-owned files points at the wrong tree.

**The reliable derivation.** `create.rs` OWNS both the write layout (it creates
worktrees at `root.join(WORKTREES_SUBDIR).join(name)`, `WORKTREES_SUBDIR =
".worktrees"`) and the coord-relative state dirs. So the reader recovers the
coord root self-consistently:
- `name = basename(cwd)`
- assert `basename(parent(cwd)) == ".worktrees"` (else it's not a dispatch
  worktree layout → fail safe)
- `coord_root = parent(parent(cwd))`
Both sides canonicalise; the write side (create-fork `root`) and the read side
(stripped cwd) agree by construction, independent of git.

**Distinct concern from worktree RECOGNITION/trust.** `cwd_is_project_worktree`
(the A1 git-topology check: `is_linked_worktree` AND common-dir == the
`CLAUDE_PROJECT_DIR` anchor's) answers *is this a trusted project worktree at
all*. That is separate from *where does this coord-owned file live* — the latter
is layout-owned and MUST use the path strip, not topology. Don't conflate them.
